# Components

## Tracker Adapters

Agent Please supports multiple issue trackers through adapter pattern:

### GitHub Projects v2

- Uses GraphQL API for project item queries
- Supports both PAT and GitHub App authentication
- Label-based status tracking (dispatched, done, failed)
- Webhook handler for real-time updates

### Asana

- Uses REST API for task queries
- PAT authentication
- Section-based status mapping
- Webhook event support

## Workspace Manager

Manages per-issue workspace directories with isolation guarantees:

- **Directory creation** — creates dedicated subdirectory under `workspace.root`
- **Git worktrees** — optional worktree isolation for GitHub-hosted repos
- **Lifecycle hooks** — `after_create`, `before_run`, `after_run`, `before_remove`
- **Path validation** — strict child-of-root check with symlink traversal prevention
- **Cleanup** — automatic cleanup of terminal-state workspaces on startup

## Agent Runner

Interfaces with Claude Code via `@anthropic-ai/claude-agent-sdk`:

- **Session management** — `startSession()` validates workspace and assigns UUID
- **Turn execution** — `runTurn()` calls `query()` with rendered prompt
- **Event translation** — converts SDK messages to orchestrator events (`session_started`, `turn_completed`, `turn_failed`)
- **MCP tool injection** — injects `github_graphql` or `asana_api` tools into agent sessions

## Config Layer

Handles WORKFLOW.md parsing and typed configuration:

- **YAML parsing** — extracts front matter from WORKFLOW.md
- **Env-var resolution** — replaces `$ENV_VAR` references with process.env values
- **Typed config** — produces `ServiceConfig` with typed getters
- **Live reload** — file watcher triggers re-parse; invalid configs rejected

## Prompt Builder

Renders agent prompts using Liquid templates:

- **LiquidJS** — Liquid-compatible template engine
- **Issue context** — `{{ issue.title }}`, `{{ issue.body }}`, etc.
- **Conditional logic** — `{% if %}` blocks for dynamic prompts

## Chat Bot

Multi-platform chat integration via Chat SDK:

- **GitHub adapter** — responds to `@mention` in issue comments
- **Slack adapter** — responds to events via webhook
- **Status responses** — running issues, retry queue, token usage
- **Lifecycle** — managed as Nitro server plugin

## Database Layer

Persistent storage for agent run history:

- **libsql/Turso** — embedded SQLite or cloud database
- **Records** — issue ID, session ID, duration, tokens, turns, status
- **Session messages** — extracted from Claude Agent SDK for replay

## Logging

Structured logging via `consola`:

- **Tag-based** — `[orchestrator]`, `[db]`, `[chat-bot]`
- **Verbose mode** — toggleable via `setVerbose()`
- **Structured output** — key=value format for parsing
