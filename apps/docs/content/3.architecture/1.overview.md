# System Overview

Agent Please is a long-running TypeScript daemon (Bun + Turborepo monorepo) that turns issue tracker tasks into autonomous Claude Code agent sessions. It follows a scheduler/runner pattern.

## Architecture

```
agent-please/                      # Monorepo root
├── apps/agent-please/             # Main Nuxt application (@pleaseai/agent)
│   ├── src/                      # CLI entry point
│   ├── app/                      # Nuxt client-side (dashboard)
│   └── server/                   # Nitro server (API, plugins, webhooks)
├── packages/core/                # @pleaseai/agent-core (business logic)
│   └── src/
│       ├── orchestrator.ts       # Core poll/dispatch/retry loop
│       ├── config.ts             # YAML front matter → typed ServiceConfig
│       ├── workflow.ts           # WORKFLOW.md parser
│       ├── prompt-builder.ts     # Liquid template rendering
│       ├── agent-runner.ts       # Claude Code agent sessions
│       ├── workspace.ts          # Per-issue directory management
│       └── tracker/              # Issue tracker adapters
└── vendor/symphony/              # Upstream reference spec (read-only)
```

## Component Boundaries

| Component | Responsibility |
|-----------|---------------|
| **Workflow Loader** | Parse WORKFLOW.md YAML front matter + Liquid prompt template |
| **Config Layer** | Typed getters with env-var indirection (`$ENV_VAR` syntax) |
| **Issue Tracker Client** | Asana REST or GitHub Projects v2 GraphQL; poll + reconcile |
| **Orchestrator** | In-memory state; poll/dispatch/retry loop |
| **Workspace Manager** | Create/reuse per-issue directories; run lifecycle hooks |
| **Agent Runner** | Invoke Claude Code via `@anthropic-ai/claude-agent-sdk` |
| **Status Surface** | Nuxt dashboard + structured logs |

## Entry Points

| File | Purpose |
|------|---------|
| `apps/agent-please/src/index.ts` | Binary entry point |
| `apps/agent-please/src/cli.ts` | CLI parsing, Nuxt server startup |
| `packages/core/src/orchestrator.ts` | Core poll/dispatch/retry loop |
| `apps/agent-please/server/plugins/01.orchestrator.ts` | Nitro plugin: creates Orchestrator |
| `apps/agent-please/server/plugins/02.chat-bot.ts` | Nitro plugin: Chat SDK bot |

## Architecture Invariants

1. **Minimal tracker writes** — the orchestrator writes only status labels. All state transitions and PR operations are performed by the agent.
2. **Workspace path validation** — every path is validated against `config.workspace.root` before agent launch.
3. **Config immutability during tick** — the config object is replaced atomically on reload, never mutated.
4. **Concurrency limits are hard** — dispatch never exceeds `max_concurrent_agents`.
5. **Error types, not exceptions** — discriminated union types for expected errors.
6. **Env-var indirection** — `$ENV_VAR` references resolved at startup.
7. **No vendor modifications** — `vendor/symphony/` is read-only.
