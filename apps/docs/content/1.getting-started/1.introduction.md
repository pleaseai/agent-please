# Introduction

Welcome to **Agent Please** — a long-running TypeScript daemon that turns issue tracker tasks into autonomous Claude Code agent sessions.

## What is Agent Please?

Agent Please enables engineering teams to manage work at scale instead of supervising individual coding agents. It continuously polls an issue tracker (GitHub Projects v2 or Asana), creates an isolated workspace for each eligible issue, renders a Liquid prompt template, and launches a Claude Code agent session inside that workspace.

## Core Value Proposition

**Zero-config issue-to-agent mapping with WORKFLOW.md as single source of truth.** A single Markdown file in the target repository defines tracker connection, agent behavior, workspace hooks, and the prompt template. No external dashboards, databases, or config servers required.

## How It Works

1. **Poll** — The orchestrator polls your issue tracker for issues in configured states
2. **Workspace** — Creates an isolated directory (or git worktree) for each issue
3. **Prompt** — Renders a Liquid template with issue context variables
4. **Agent** — Launches a Claude Code session via `@anthropic-ai/claude-agent-sdk`
5. **Retry** — On failure, applies exponential backoff; on clean exit, schedules continuation

The service is primarily a **scheduler/runner** — it writes only status labels to the tracker. All state transitions, PR creation, and comments are performed by the Claude Code agent itself.

## Key Capabilities

- **Poll-based orchestration** with configurable intervals
- **WORKFLOW.md configuration** — version agent prompt and settings alongside your code
- **Workspace isolation** with lifecycle hooks (`after_create`, `before_run`, `after_run`, `before_remove`)
- **Bounded concurrency** — global and per-state agent limits
- **Retry with exponential backoff** on failures
- **Multi-tracker support** — GitHub Projects v2 and Asana
- **Chat bot integration** — GitHub issue comments and Slack via Chat SDK
- **Dashboard** — Nuxt-based HTTP dashboard with real-time state

## What Agent Please is NOT

- **Not a general-purpose AI agent framework** — focused on the issue tracker to coding agent pipeline
- **Not a code review tool** — the agent performs implementation; review remains separate
- **Not a CI/CD replacement** — complements existing pipelines
